# Отчет об ошибках (Bug Reports)

### Bug-01: Некорректная работа фильтра максимальной цены на Главной странице

**Краткое описание:** При фильтрации объявлений по диапазону цен, в поисковую выдачу попадают карточки товаров, цена которых превышает установленное максимальное значение.

**Шаги воспроизведения:**
1. Открыть главную страницу платформы.
2. В блоке фильтров ввести в поле «От» значение `1000`.
3. Ввести в поле «До» значение `8000`.
4. Дождаться обновления списка объявлений.
5. Проверить цены в отображаемых карточках.

**Фактический результат:** В выдаче присутствуют объявления с ценой, выходящей за верхнюю границу фильтра (например, отображается товар с ценой 9905 ₽ при максимальном лимите в 8 000 ₽). Тест `testPriceFilter` падает с ошибкой: `java.lang.AssertionError: Ошибка: Цена 9905 больше максимальной 8000`.

**Ожидаемый результат:** В списке отображаются только те объявления, цена которых строго меньше или равна 8 000 ₽ и больше или равна 1 000 ₽.

**Серьезность (Severity):** High / P1 (Нарушена базовая логика поиска).
**Окружение:** ОС Linux (Ubuntu), браузер Firefox, Desktop.

### Bug-02: Не работает фильтрация объявлений при активации тогла «Только срочные»

**Краткое описание:** При включении переключателя «Только срочные» (или выборе опции «Срочный» в фильтре Приоритет) список объявлений не обновляется, в выдаче остаются обычные (не срочные) объявления.

**Шаги воспроизведения:**
1. Открыть главную страницу платформы.
2. Нажать на переключатель (тогл) «Только срочные» в панели фильтров.
3. (Опционально) Убедиться, что в скрытом селекте «Приоритет» значение изменилось на `urgent`.
4. Проверить список отображаемых объявлений.

**Фактический результат:** Запрос на фильтрацию не отрабатывает (список не обновляется). В выдаче продолжают отображаться все объявления, включая те, у которых нет статуса срочности.

**Ожидаемый результат:** Список объявлений должен мгновенно обновиться, оставив в выдаче только те карточки, которые имеют маркер/статус «Срочное».

**Серьезность (Severity):** High / P1 (Не работает заявленный функционал фильтрации).

### Bug-03: Не работает кнопка возврата к списку объявлений

**Краткое описание:** При нажатии на кнопку «Назад к списку» внутри карточки товара переход на главную страницу не происходит.
**Шаги воспроизведения:**
1. Открыть главную страницу.
2. Кликнуть на любую карточку товара для перехода в детальный просмотр.
3. Нажать кнопку «Назад к списку».
**Фактический результат:** Страница не меняется, пользователь остается в карточке товара. URL не изменяется.
**Ожидаемый результат:** Происходит переход на главную страницу, отображается список всех объявлений.
**Severity:** Critical (Блокирует навигацию пользователя).

### Bug-04: Не работает кнопка «Сбросить фильтры»

**Краткое описание:** После применения фильтров (цена, категория) нажатие на кнопку «Сбросить» не возвращает значения фильтров и список товаров в исходное состояние.
**Шаги воспроизведения:**
1. Выбрать категорию или ввести диапазон цен.
2. Дождаться обновления списка товаров.
3. Нажать кнопку «Сбросить фильтры».
**Фактический результат:** Значения в полях ввода и селектах остаются прежними, список товаров не сбрасывается до полного.
**Ожидаемый результат:** Все поля фильтров очищаются, селекты возвращаются в дефолтное состояние, на странице снова отображаются все доступные товары.
**Severity:** Major (Ломает логику работы с поисковой выдачей).

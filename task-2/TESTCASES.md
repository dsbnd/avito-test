# Тест-кейсы: Платформа модерации объявлений (UI)

В данном документе описаны тест-кейсы для проверки пользовательских сценариев платформы. Учитывая динамический характер тестового стенда, проверки реализованы без привязки к конкретным тестовым данным или фиксированному количеству элементов.

## Десктопная версия

### TC-01: Проверка работы фильтра «Диапазон цен» (Главная страница)
**Шаги:**
1. Открыть главную страницу платформы.
2. В блоке фильтров ввести минимальную цену (например, 100).
3. Ввести максимальную цену (например, 5000).
4. Дождаться применения фильтра (обновления списка объявлений).
5. Собрать цены всех отображаемых на странице объявлений.

**Ожидаемый результат:** Цена каждого объявления на странице больше или равна минимальной и меньше или равна максимальной заданной цене.

### TC-02: Проверка сортировки «По цене» (Главная страница)
**Шаги:**
1. Открыть главную страницу.
2. В выпадающем списке сортировки выбрать «По убыванию» (или «По возрастанию»).
3. Дождаться обновления списка объявлений.
4. Собрать цены всех отображаемых объявлений в массив.

**Ожидаемый результат:**
Массив цен строго отсортирован в выбранном порядке (каждый следующий элемент меньше/равен или больше/равен предыдущему).

### TC-03: Проверка работы фильтра «Категория» (Главная страница)
**Шаги:**
1. Открыть главную страницу.
2. Выбрать любую доступную категорию в фильтре (например, «Авто»).
3. Дождаться обновления списка.
4. Проверить атрибуты/теги всех отображаемых объявлений.

**Ожидаемый результат:**
У всех карточек объявлений на странице присутствует маркер выбранной категории.

### TC-04: Проверка работы тогла «Только срочные» (Главная страница)
**Шаги:**
1. Открыть главную страницу.
2. Активировать переключатель «Только срочные».
3. Дождаться обновления списка.
4. Проверить наличие бейджа срочности у всех карточек.

**Ожидаемый результат:**
Каждое объявление в поисковой выдаче содержит бейдж/иконку, обозначающую срочность. Ни одного обычного объявления нет.

### TC-05: Управление таймером обновления (Страница Статистики)
**Шаги:**
1. Переключиться на страницу `/stats`.
2. Зафиксировать текущее время на таймере и нажать кнопку «Остановить».
3. Подождать 3 секунды и проверить значение таймера.
4. Нажать кнопку «Запустить таймер» и подождать 3 секунды.
5. Нажать кнопку «Обновить».

**Ожидаемый результат:**
* После шага 3: Таймер не изменяется (поставлен на паузу).
* После шага 4: Таймер возобновил обратный отсчет.
* После шага 5: Данные статистики обновлены, таймер сброшен на начальное значение.

## Мобильная версия

### TC-06: Переключение между тёмной и светлой темой
**Предусловие:** Запустить браузер с эмуляцией мобильного разрешения (например, 390x844).
**Шаги:**
1. Открыть главную страницу.
2. Найти и нажать кнопку переключения темы (луна/солнце).
3. Проверить изменения в DOM-дереве (например, появление атрибута `data-theme="dark"` у тега `<body>` или изменение цвета фона).
4. Нажать кнопку переключения темы повторно.
   
**Ожидаемый результат:**
Стиль приложения корректно переключается на темную тему и обратно на светлую. Цвета фона и текста соответствуют выбранной теме.

# Task 2: UI Automation (Selenium + Java)

В данном модуле реализованы автоматизированные сценарии для проверки стабильности работы веб-приложения на общем тестовом стенде.

Основные разработанные тест-кейсы в [`TESTCASES.md`](./TESTCASES.md)

Найденные баги в [`BUGS.md`](./BUGS.md)

## Инструкция по запуску

### Предварительные требования
1. Установленный браузер **Firefox**.
2. Переменная окружения `JAVA_HOME` указывает на JDK 17.

### Запуск
Перейдите в папку `task-2` и выполните команду:
```bash
mvn clean test
```

## Стек технологий и версии
* **Java:** 17
* **Build Tool:** Maven
* **Testing Framework:** TestNG 7.9.0
* **Browser Automation:** Selenium WebDriver 4.18.1
* **Browser:** Firefox (GeckoDriver)

## Реализованные сценарии (6 тестов)
1.  **Фильтрация по цене** — проверка корректности отображения товаров в заданном диапазоне.
2.  **Сортировка по цене** — проверка работы алгоритма сортировки по возрастанию.
3.  **Фильтрация по категориям** — динамический выбор категории и валидация выдачи.
4.  **Тогл "Только срочные"** — проверка фильтрации объявлений со специальным статусом.
5.  **Логика таймера (Статистика)** — проверка запуска, паузы и сброса счетчика автообновления.
6.  **Мобильная адаптивность** — проверка смены темы оформления (Dark/Light) на мобильном разрешении экрана.

## Особенности реализации
* **Устойчивость к данным:** Тесты адаптированы к "шумному" общему стенду. Используются динамические локаторы и `WebDriverWait` с лямбда-выражениями, что исключает падения при изменении количества товаров.
* **Clean Code:** Применение Java Stream API для валидации коллекций элементов.
* **No Hardcoded Sleeps:** Вместо статических пауз используются ожидания изменения состояний элементов (текста, атрибутов).
